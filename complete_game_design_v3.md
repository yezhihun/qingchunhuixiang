# 青春回响 - 完整游戏设计方案 V3

## 核心定位
**类型**：高中生活模拟 + 情感共鸣 + 社交分享  
**平台**：微信小游戏  
**核心体验**：让玩家重回高中时代，通过观察和影响AI学生的生活，产生情感共鸣并愿意分享故事切片

## 关键决策确认
- **互动深度**：预设选项（确保体验一致性和内容质量）
- **影响范围**：免费关注主要学生，付费解锁观察其他同学  
- **分享内容**：纯游戏剧情/故事切片，无隐私问题
- **美术风格**：简笔画 + 怀旧风格（低成本高共鸣）
- **时间同步**：完全同步现实时间（增强真实感）
- **商业化**：暂不考虑，专注核心体验

## 系统架构设计

### 1. 学生系统
**学生数量**：40-45人/班级（标准高中班级规模）  
**学生画像**：
- 基础信息：姓名、年龄、性别、身高、体重、形象特征
- 性格特征：Big Five性格模型 + 具体特质标签  
- 背景故事：家庭情况、学术历史、社交经历、高考分数、高光时刻
- 能力属性：学业、体育、社交、创造力、情商五大维度
- 实时状态：精力、压力、心情、专注度、动机水平

**多样性设计**：
- 学业导向型：学霸、努力型普通生、偏科生
- 社交导向型：社交达人、小团体核心、孤独观察者  
- 问题学生类型：叛逆反抗型、消极逃避型、欺凌者、被欺凌者、早恋问题生、家庭问题生
- 特殊情况：转学生、复读生、体育/艺术特长生

### 2. 时间系统
**混合时间模型**：固定课表框架 + 动态事件覆盖

**基础时间结构**：
- 学年 → 学期 → 周 → 日 → 时段
- 固定课表：8节课/天，标准高中作息时间
- 现实时间完全同步

**动态覆盖层**：
- 拖堂事件（概率15%，5-20分钟）
- 班主任占课（特定老师高概率）
- 特殊事件：集会、考试、运动会等

**年度事件系统**：
- **考试体系**：日常测验、月考、期中期末、体育考试
- **重大活动**：军训（开学第一周）、运动会（10月中旬）、联欢晚会（12月31日）、毕业典礼（7月）
- **节假日**：寒暑假、国庆、五一、清明等传统节日
- **寒暑假子系统**：轻度互动维持玩家连接

### 3. 场景系统
**动态场景架构**：
- 基础场景：教室、走廊、食堂、操场、图书馆
- 动态元素：黑板内容、桌椅排列、墙面展示、临时道具
- 展示方案：静态背景 + 动态UI元素叠加

**美术风格**：简笔画 + 怀旧风格
- 低成本实现高情感共鸣
- 符合微信小游戏包大小限制

### 4. 事件系统
**配置驱动架构**：
- 所有事件通过JSON配置文件管理
- 支持热更新，无需重新部署

**分层事件设计**：
- **Layer 1: 日常剧情**（高频，轻量）- 课堂插曲、课间对话
- **Layer 2: 个人主线**（中频，中等深度）- 每个学生的核心成长故事  
- **Layer 3: 班级大事件**（低频，高深度）- 运动会、联欢晚会、重大考试
- **Layer 4: 隐藏剧情**（稀有，高价值）- 特殊条件触发的深度故事

**触发机制**：时间条件 + 状态条件 + 关系条件 + 玩家行为

### 5. AI决策引擎
**规则驱动架构**：
- 基于预设规则的行为选择，非复杂AI
- 插件化决策模块，支持扩展

**决策分层模型**：
- Layer 1: 基础需求驱动（生理、安全、社交）
- Layer 2: 性格特征过滤（Big Five影响权重）
- Layer 3: 环境约束评估（场景规则、时间限制、社会压力）
- Layer 4: 目标导向选择（短期/长期目标、风险收益）

### 6. 玩家互动系统
**互动模式**：观察 + 轻度影响
- **免费功能**：关注1-2个主要学生，预设选项互动
- **付费扩展**：解锁观察其他同学，获得更多互动选项

**核心循环**：每日观察 → 选择互动 → 看结果 → 获得反馈 → 计划明天

**预设选项设计**：
- 每个学生5-8个核心互动选项
- 选项基于学生性格和当前状态动态调整
- 确保每个选择都有明确的后果和意义

### 7. 分享系统
**分享类型**：
- **成就分享**："我的同桌李明终于在数学考试中及格了！🎉"
- **故事分享**："今天张伟偷偷给暗恋的女孩送了奶茶，结果被全班起哄..."
- **感悟分享**："高中三年，最珍贵的不是成绩，而是那些陪你一起疯的朋友 ✨"

**分享格式**：图片卡片 + 短文本（自动美化排版）
**隐私保护**：纯游戏内虚拟角色故事，无真实隐私问题
**用户控制**：完全由玩家自主决定是否分享，无强制推送

## 主要风险与缓解策略

### 技术风险
1. **包大小超限**（12MB限制）
   - 缓解：简笔画风格、高效数据结构、按需加载
   
2. **性能瓶颈**（40学生状态计算）
   - 缓解：离散事件系统、懒加载、批量更新

3. **存储限制**（本地10MB存储）
   - 缓解：云端备份重要数据、数据压缩

### 内容风险
1. **工作量巨大**（高质量剧情内容）
   - 缓解：模板化生成 + 人工精修关键内容

2. **质量一致性**（情感深度保证）
   - 缓解：建立内容质量标准、用户测试验证

### 产品风险
1. **玩法粘性**（观察型游戏留存）
   - 缓解：强化情感共鸣、增加轻度策略元素

2. **市场差异化**（与现有校园游戏区别）
   - 缓解：聚焦真实高中生活体验、强调情感共鸣
